# Recap Bot

Telegram-бот делает короткий живой пересказ русскоязычного группового чата. Он хранит сообщения последних 24 часов в памяти, умеет начинать рекап с сообщения, на которое ответили командой `/recap`, расшифровывает голосовые сообщения и кружочки, а затем обращается к OpenAI-совместимому API.

При наличии PostgreSQL с расширением pgvector бот дополнительно сохраняет всю историю в базе данных, индексирует её в виде смысловых фрагментов с векторными эмбеддингами и поддерживает семантический поиск командами `/search`, `/s`, `/п` и префиксом `?`.

## Возможности

- `/recap` — пересказ доступной переписки за последние сутки.
- `/recap` ответом на сообщение — пересказ начиная с этого сообщения.
- `/help` — краткая инструкция.
- `/search <запрос>`, `/s <запрос>`, `/п <запрос>` или `? <запрос>` — семантический поиск по истории текущего чата (только при PostgreSQL).
- `/init` — загрузить экспорт Telegram Desktop (`result.json`, `.json.gz` или `.zip`) для импорта истории (только администраторы чата).
- `/init_status` — показать количество сообщений и фрагментов в базе данных.
- `/init_cancel` — отменить текущий импорт.
- Расшифровка голосовых сообщений и видеокружков.
- Описание изображений: одиночное фото — через BIG-модель, альбом (до 3 фото) — через SIMPLE-модель.
- Безопасная проверка созданных моделью ссылок на сообщения.

История хранится в оперативной памяти (последние 24 часа) и при наличии PostgreSQL — постоянно в базе данных. Для чтения обычных сообщений в группах отключите privacy mode через BotFather, а затем заново добавьте бота в группу.

## Установка и запуск

Требуется Python 3.10 или новее.

```bash
cd recap
python -m venv .venv
source .venv/Scripts/activate
pip install -r requirements.txt
```

### Только память (без PostgreSQL)

Задайте обязательные переменные окружения:

```
TELEGRAM_BOT_TOKEN=...
OPENAI_TOKEN=...
```

```bash
python recap/recap.py
```

### С PostgreSQL и pgvector

Установите PostgreSQL 14+ и расширение pgvector:

```bash
# Ubuntu / Debian
sudo apt install postgresql postgresql-contrib
sudo -u postgres psql -c "CREATE EXTENSION vector;"

# macOS (Homebrew)
brew install pgvector
```

Создайте базу данных и пользователя:

```sql
CREATE USER recap_user WITH PASSWORD 'secret';
CREATE DATABASE recap OWNER recap_user;
\c recap
CREATE EXTENSION IF NOT EXISTS vector;
```

Задайте переменные окружения (таблицы создаются автоматически при первом запуске):

```
PG_DATABASE=recap
PG_USERNAME=recap_user
PG_PASSWORD=secret
PG_HOST=localhost
PG_PORT=5432
```

### Обязательные

| Переменная | Описание |
|---|---|
| `TELEGRAM_BOT_TOKEN` | Токен Telegram-бота |
| `OPENAI_TOKEN` | Ключ OpenAI-совместимого API |

### Модели и API

| Переменная | По умолчанию | Описание |
|---|---|---|
| `RECAP_LLM_URL` | `https://api.openai.com/v1` | URL OpenAI-совместимого API |
| `RECAP_OPENAI_MODEL` | `google/gemini-2.5-flash-lite` | Модель для рекапа, поиска и индексации |
| `RECAP_TRANSCRIPTION_MODEL` | `openai/whisper-large-v3` | Модель распознавания речи |
| `RECAP_TRANSCRIPTION_LANGUAGE` | — | Язык распознавания, например `ru` |
| `RECAP_IMAGE_MODEL_BIG` | `google/gemini-3.1-flash-lite` | Модель для одиночных изображений |
| `RECAP_IMAGE_MODEL_SIMPLE` | `google/gemini-2.5-flash-lite` | Модель для альбомов (2–3 фото) |
| `RECAP_EMBEDDING_MODEL` | `openai/text-embedding-3-small` | Модель для векторных эмбеддингов |
| `RECAP_EMBEDDING_DIM` | `1536` | Размерность эмбеддинга |

### PostgreSQL (опционально)

| Переменная | По умолчанию | Описание |
|---|---|---|
| `PG_DATABASE` | — | Имя базы данных (наличие активирует персистентное хранение) |
| `PG_USERNAME` | — | Пользователь базы данных |
| `PG_PASSWORD` | — | Пароль |
| `PG_HOST` | `localhost` | Хост |

| Переменная | По умолчанию | Описание |

## Семантический поиск (/search, /s, /п, ?)

При наличии PostgreSQL бот:

1. Нормализует запрос через LLM (извлекает фильтры по дате, участникам и точным фразам).
2. Строит векторный эмбеддинг нормализованного запроса.
3. Выполняет гибридный поиск: векторный (cosine) + полнотекстовый (tsvector), объединяет результаты через Reciprocal Rank Fusion.
4. Опционально ранжирует кандидатов через LLM.
5. Загружает сообщения лучшего фрагмента и генерирует грамотный ответ.
6. Отправляет ответ репляем на сообщение, на которое ссылается ответ (сначала пробует процитированные в ответе id, затем важные сообщения фрагмента, затем первое сообщение, затем любое сообщение фрагмента) — старые/импортированные сообщения могут быть удалены из живого чата, поэтому пробуется несколько вариантов, прежде чем отправить без реплая.

Все ссылки на сообщения строятся приложением (не LLM) и проверяются перед отправкой.

⚠️ Стабильные ссылки на сообщения (`t.me/c/...` или `t.me/<username>/...`) существуют в Telegram только для супергрупп/каналов. В обычной (не супер) группе без публичного username ссылку построить невозможно — в этом случае вместо `<a href>` в ответе показывается обычный текстовый маркер `(#N)`, а связь с источником обеспечивается только через реплай.

## Импорт истории (/init)

Позволяет загрузить экспорт Telegram Desktop для заполнения исторической базы данных.

1. Только администратор или владелец чата может запустить `/init`.
2. Бот ожидает файл: `result.json`, `result.json.gz` или ZIP-архив с экспортом.
3. Файл разбирается потоком через `ijson` и нормализуется.
4. ⚠️ После успешного разбора файла вся ранее сохранённая история этого чата (сообщения и фрагменты) удаляется, и импортированные сообщения загружаются в базу данных с нуля — это гарантирует полную переиндексацию без смешивания со старыми/частичными данными.
5. Прогресс показывается редактированием статусного сообщения.
6. `/init_cancel` останавливает импорт; уже загруженные записи сохраняются, история не восстанавливается.
7. После импорта фоновый индексатор автоматически обработает новые сообщения (индексация идёт непрерывно, пока есть очередь).

```bash
pytest recap/
```
